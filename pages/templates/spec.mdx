# Spec template

The 8 canonical sections every spec should contain. Select the code block below and paste it into `specs/[your-feature]/spec.md` in your project.

````markdown
# Spec — [Feature name]

> **Status**: draft | active | archived
> **Author**: [your name]
> **Created**: YYYY-MM-DD
> **Last updated**: YYYY-MM-DD

---

## 1. Problem

One concrete paragraph. What's broken or missing, and what's the *consequence* of not solving it.

## 2. Audience

| Persona | Need | Frequency |
|---|---|---|
| | | |

## 3. User stories

### US-1 — [short title]
> *As a [persona]*, I want [capability], *so that* [outcome].

## 4. Acceptance criteria (AC)

| # | Criterion | Testable by |
|---|---|---|
| AC-1 | | |

## 5. Constraints

## 6. Non-goals

## 7. Success metrics

## 8. Risks
````
