# Getting started

This guide gets you from zero to your first `spec.md / plan.md / tasks.md` trio in **under 15 minutes**, on a real project.

## Prerequisites

- Any JavaScript/TypeScript project (new or existing)
- A CLI agent (we use **Claude Code**, but patterns translate to other agents)
- Basic Markdown comfort

## The 3-file convention

Every feature, however small, lives in `specs/[feature-slug]/` with three files:

```
specs/
└── my-feature/
    ├── spec.md     # what & why
    ├── plan.md     # how
    └── tasks.md    # atomic steps
```

<Callout type="info">
  One folder per feature. They live alongside your code, commit with it, PR with it.
</Callout>

## Install the conventions

<Steps>

### Create the specs directory

```bash
mkdir -p specs
echo "# Specs\n\nSpec-driven development artefacts." > specs/README.md
```

### Grab the starter templates

Open each template page, select the code block, and paste it into your own `specs/` folder:

- [Spec template](/templates/spec) → save as `specs/TEMPLATE-spec.md`
- [Plan template](/templates/plan) → save as `specs/TEMPLATE-plan.md`
- [Tasks template](/templates/tasks) → save as `specs/TEMPLATE-tasks.md`

Each template page shows the full Markdown — use your editor's copy button on the code block.

### Start your first feature

```bash
mkdir -p specs/my-first-feature
cp specs/TEMPLATE-spec.md specs/my-first-feature/spec.md
```

Open `spec.md` and fill the sections in order: problem, audience, user stories, acceptance criteria.

### Hand it to your agent

In your Claude Code session:

> Read `specs/my-first-feature/spec.md` and generate a `plan.md` respecting the stack conventions in `CLAUDE.md`.

Then:

> From `plan.md`, generate `tasks.md` with atomic tasks. Each task must reference an AC.

### Iterate

Review, correct, commit. **The specs become the source of truth** for future agent sessions.

</Steps>

## What success looks like

After 15 minutes you should have:

- ✅ `specs/my-first-feature/spec.md` with 5-10 user stories and 5+ ACs
- ✅ `specs/my-first-feature/plan.md` with stack choices explicitly argued
- ✅ `specs/my-first-feature/tasks.md` with 10-30 atomic tasks

From there, let your agent burn through the task list — each completion is a small commit.
