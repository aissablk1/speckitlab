## Why another SDD resource?

The [official Spec-Kit](https://github.com/github/spec-kit) is Python-centric. Most SDD content online is textual, dispersed, or tied to a single agent framework. speckitlab fills that gap for JavaScript/TypeScript teams who want:

- A **single hub** with templates they can copy-paste today
- **Real examples** (not toy projects) showing the `spec → plan → tasks` flow
- A clear **Claude Code integration** guide, because that's where SDD becomes a superpower

## The 3-artefact loop

<Steps>

### `spec.md` — the *what*

User stories, acceptance criteria, non-goals, metrics. Survives refactors.

### `plan.md` — the *how*

Stack choices, architecture, data contracts, security posture. Evolves with the project.

### `tasks.md` — the *steps*

Atomic, verifiable, traceable back to an acceptance criterion. This is what you hand to Claude Code.

</Steps>
